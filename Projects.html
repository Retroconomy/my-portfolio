About This Project

This is my Pokémon Information System, also known as the Pokedex. It is a tool designed to showcase the information of all Pokémon from the very first generation. By leveraging a custom API, I connected a database that retrieves and displays Pokémon information, including their names, types, and other essential details.

How I Built It

The creation of this Pokedex required multiple steps:

  • Custom API Development: I designed a custom API that interacts with a Pokémon database, fetching data about each Pokémon from Generation 1.
  • Database Integration: The database stores critical information about each Pokémon, such as their types, evolutions, abilities, and base stats. The API queries this data based on the user’s input.
  • Frontend Design: Using HTML, CSS, and JavaScript, I created a user-friendly interface that displays Pokémon information in a clear and appealing way. Each Pokémon is represented with a sprite, its name, and its types.

How to Make Your Own Pokedex

If you'd like to create a similar Pokedex, here are the basic steps:

  1. Gather Pokémon Data: Use a Pokémon API like PokeAPI or create your own database of Pokémon information.
  2. Set Up a Backend: Build a server-side application (e.g., Node.js) that fetches and serves Pokémon data.
  3. Design the Interface: Use HTML and CSS to design a simple, intuitive interface for the user.
  4. Add Search/Filter Features: Implement JavaScript to allow users to search for specific Pokémon or filter them by type or generation.
  5. Host Your Application: Deploy your project using services like GitHub Pages, Netlify, or Heroku to make it accessible online.

Technologies Used

For this project, I used the following technologies:

  • HTML, CSS, JavaScript: For the user interface and interactivity.
  • Custom API: Built using Node.js and Express to fetch data dynamically.
  • Database: MySQL for storing detailed Pokémon information.
